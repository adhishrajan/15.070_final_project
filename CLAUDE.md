## Key Design Principle: Tractability and Fair Comparison

Brute force is O(k³·C(n,k)) — only tractable for C(n,k) ≲ 10⁵ (n ≲ 30 at k=5).

**Rule:** Any experiment that uses brute force as the exact benchmark must evaluate ALL methods (BF, Greedy, SA, and the Markowitz UB) on the **same** n-stock sub-universe. Using a different universe for BF vs. Greedy/SA would be an unfair comparison.

**Consequence:**
- **Exp B** (quality comparison): all 4 methods use the same 30 curated stocks (BF_TICKERS). The Markowitz UB of 47.79% is computed on this 30-stock subset, NOT on N=462.
- **Exp C, D** (scaling / cost of cardinality): no exact benchmark needed → Greedy and SA run on full N=462.
- **Exp E, F** (constraint tightness / random sampling): use 35 curated stocks (EF_TICKERS) — brute-force pre-computation of all C(35,5)=324,632 subsets is tractable.

## Core Model: Short Selling Allowed (no w ≥ 0)

**Decision**: weights are unconstrained — `w ∈ ℝᴺ`, short positions permitted.

**Consequence**: the inner problem for each k-subset is exactly the classical
Markowitz mean-variance problem, which has a **closed-form solution** (no iterative
solver needed):

```
A = 1ᵀ Σ⁻¹ 1,   B = 1ᵀ Σ⁻¹ μ,   C = μᵀ Σ⁻¹ μ,   D = AC - B²

σ²_min = 1/A                                  (global min-variance for the subset)
μ*     = B/A + sqrt(D/A * (σ₀² - 1/A))       (optimal return — closed form)
w*     = (1/D) * [(C - B·μ*) Σ⁻¹1 + (A·μ* - B) Σ⁻¹μ]   (two-fund weights)
```

Infeasible iff σ₀² < 1/A (risk budget below min-variance of the subset).
Also returns (None, None) if A < 1e-14 (near-singular Σ_S).

**`socp_upper_bound` is just `socp_inner` applied to all N assets** — cvxpy is
not used anywhere in the notebook.

---

## Critical Design Decision: SIGMA_SQ_MAX

**Correct approach** (current code):
```python
var_individual = np.diag(Sigma_all)
SIGMA_SQ_MAX = np.percentile(var_individual, 25)   # = (22.80%)² annualised
```

Setting σ₀ = 25th percentile of individual stock vols (~22.80% here):
- ~75% of stocks have vol > σ₀ (constraint is genuinely binding)
- 97.3% of all 5-subsets are feasible (with short selling helping achieve lower variance)
- The full N-asset Markowitz problem is feasible

**Wrong approach** (old code): `SIGMA_SQ_MAX = 0.60 * var_mkt` where `var_mkt`
= equal-weight N-stock portfolio variance. The diversified portfolio has vol ≈ 17.44%,
so this set σ₀ ≈ 13.5% — far below any k-subset's min-variance, making all inner
problems infeasible and all algorithms return -inf.

## Bugs Fixed (history)

### ZeroDivisionError in socp_inner (cell-21, degenerate branch)
In `if D < 1e-14`, code did `sigma_sq_min = 1.0 / A` without checking A.
During the Exp E constraint sweep, some sub-covariance matrices are near-singular,
giving A ≈ 0 and crashing with ZeroDivisionError.
Fixed by adding a guard before the degenerate branch:
```python
if A < 1e-14:
    return None, None
```

### TypeError: unsupported operand for -: NoneType and float (cell-15)
`socp_upper_bound` returns `(None, None)` when infeasible.
Fixed by guarding: `if ub_ret_small is not None and bf_valid: card_gap = ...`

### ValueError: matmul 0 dimensions (cell-9)
`w_ub = None` coerced to 0-d array by NumPy in `@` operator.
Fixed by: `if w_ub is None: print diagnostic; else: compute vol_ub`.

### ValueError: 'a' cannot be empty (simulated_annealing, cell-11)
When `k >= n`, `not_in = []` and `np.random.choice(not_in)` crashes.
Fixed by early return at top of SA + `if not not_in: break` inside loop.

### Index out-of-bounds in Experiments C, D, E, F
Old code hardcoded n values that exceeded N=30.
Fixed: all experiments now cap at `min(N, x)` and filter k < n.

### Summary crash on None/-inf (cell-25)
`f"{None:.4%}"` raises TypeError.
Fixed with `_fmt()` / `_fmt_gap()` helpers returning "N/A" for non-finite values.
